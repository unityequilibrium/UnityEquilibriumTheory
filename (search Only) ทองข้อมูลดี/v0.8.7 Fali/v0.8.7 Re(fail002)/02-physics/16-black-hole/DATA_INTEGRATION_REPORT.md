## 1. Incident Summary

During the initial execution of **Phase 17 (Black Hole Integration)**, the test runner failed with a `pandas.errors.ParserError`.

### Error Log
```text
pandas.errors.ParserError: Error tokenizing data. C error: Expected 2 fields in line 4, saw 6
```
This prevented the validation script from loading the real-world elliptical galaxy data, causing the test to either crash or fallback to "Theory Only" mode (depending on exception handling).

## 2. Root Cause Analysis

The source dataset (`black_hole_sample.csv`) contains metadata headers formatted as comments:

```csv
# Kormendy & Ho (2013) Table 2 - Elliptical Galaxies (Sample)
# Source: ARAA 51, 511-653
# log_MBH and log_Mbulge in solar masses
name,log_MBH,log_Mbulge,log_ratio,distance_Mpc,notes
NGC4889,10.32,12.16,-1.840,103.0,Coma cluster, largest known BH
...
```

**The Issue:**
By default, `pandas.read_csv()` treats the first line as the header and attempts to infer the number of columns.
1. It read line 1 (`# Kormendy...`) as a single column (or inferred separator).
2. It encountered line 4 (`name,log_MBH...`) which has 6 columns.
3. This mismatch triggered the `Expected 2 fields... saw 6` error.

## 3. Technical Resolution

We modified the data loading logic in `research/run_unified_tests.py` to explicitly handle comment lines.

**Code Change:**
```python
# Before
df = pd.read_csv(csv_path)

# After
df = pd.read_csv(csv_path, comment="#")
```

The `comment="#"` argument instructs the CSV parser to:
1. Ignore any lines starting with `#`.
2. Treat the first non-comment line (`name,log_MBH...`) as the true header.
3. Correctly parse the 6-column structure.
